								Each rule set is a saved, named filter, scoped to one endpoint — a rule set built for Radarr's endpoint can't be selected for a Sonarr fetch, and vice versa. Build as many rule sets as you need per endpoint and pick the right one from a fetch's "Rule set" field on the Folder Tree tab. Editing above always edits whichever rule set is selected in the bar; nothing is saved until you click Save.
							

							

								Drag pieces from the palette onto the canvas to build the rule. Fields and operators drop into their matching slot on a condition. Drag "Condition" or "Group ( )" into a group to add to it. Drag AND / OR onto a group's connector badge, and NOT onto a condition or group to negate it. Boolean fields get a fixed True/False toggle instead of a typed value.
							

						

					

				

			


			
			

				

					Build the folder structure users browse in the Sync Channel. Each folder (including the root) can have one or more fetches — each fetch names a Connection, an Endpoint, and a Rule set. Saving is blocked only if a fetch references a connection, endpoint, or rule set that no longer exists at all — those are highlighted in red below, with the specific problem named, so you know exactly what to fix. A connection or rule set that currently exists but is unreachable is not blocked; it will simply be skipped (and logged) until it's reachable again. Saving the folder tree, a connection, or a rule set all trigger an immediate re-sync of whatever's affected — no need to run "Sync Coming Soon Folder Tree" from Scheduled Tasks manually.
